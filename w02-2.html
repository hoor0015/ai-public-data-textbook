2주차 2회차. 에이전트가 일하는 모습 관찰하기

이 실습에서 하는 것

1회차에서 에이전트의 속(LLM, 맥락창, 환각, 에이전트 루프, 도구)을 원리로 배웠다. 이번 실습의 목표는 그 원리들을 전부 내 눈으로 직접 목격하는 것이다. 오늘은 분석 결과를 만드는 날이 아니라, 에이전트라는 도구의 성질을 실험하는 날이다. 과학 시간의 관찰 실험처럼, 예상을 적고 → 실험하고 → 관찰을 기록한다.

이 실습이 끝나면 다음을 직접 확인하게 된다.

  • 에이전트 루프(계획 → 행동 → 관찰)가 대화창 로그에 그대로 드러난다는 것
  • 없는 것을 물으면 정말로 환각이 나오는지, 어떻게 물으면 줄어드는지
  • 같은 지시에 매번 조금씩 다른 답이 나온다는 것 (비결정성)
  • 승인(권한) 요청이 언제 나타나고 왜 필요한지

아직 파이썬 환경(4주차)을 갖추기 전이므로, 오늘 에이전트가 하는 계산의 정밀함보다는 일하는 방식 자체에 집중한다.

준비물

구분 내용
폴더 1주차에 만든 공공데이터분석 폴더
데이터 수업 자료실에서 받은 sigungu_2023.csv를 data 폴더에 넣어 둘 것 (전국 229개 시군구의 인구 지표, 출처: 행정안전부 주민등록인구통계·국가데이터처(옛 통계청) 인구동향조사, KOSIS에서 수집)
선수 내용 2주차 1회차 (에이전트 루프, 환각, 맥락창)

2.1 요약 작업을 시키고 도구 호출 로그 읽기

첫 실험은 평범한 요약 작업이다. 다만 오늘은 결과가 아니라 과정을 본다.

지시문 예시
"data 폴더의 sigungu_2023.csv 파일을 살펴보고, (1) 몇 개 시군구가 들어 있는지, (2) 어떤 항목(열)들이 있는지, (3) 총인구가 가장 많은 시군구 3곳과 가장 적은 시군구 3곳을 알려 줘. 어떤 방법으로 확인했는지도 함께 설명해 줘."

예상되는 에이전트의 행동. 대화창에 에이전트의 작업이 단계별로 표시된다. 파일을 읽는 단계, 내용을 확인하고 세는 단계(간단한 명령을 실행하겠다며 승인을 구할 수 있다), 그리고 결과 보고. 1회차 그림 2-3의 루프가 화면에 그대로 펼쳐지는 셈이다.

관찰 과제. 대화창을 위로 스크롤하며 다음 질문에 답해 보라.

  1. 에이전트가 가장 먼저 한 행동은 무엇인가. (바로 답했는가, 파일부터 열었는가)
  2. 도구 사용이 몇 번 있었는가. 각각 무엇을 하는 행동이었는가.
  3. 에이전트가 중간에 계획을 바꾸거나 재시도한 흔적이 있는가.

검증. 보고된 숫자 중 하나를 손으로 확인한다. 탐색기에서 sigungu_2023.csv를 클릭해 열고(VSCode에서 표 형태로 보인다), "총인구가 가장 많다"고 보고된 시군구를 찾아 값을 눈으로 대조하라. 파일을 직접 열어 볼 수 있다는 것, 이것이 여러분의 검증 수단이다.

2.2 일부러 환각 유도하기: 없는 통계를 물어보면

이제 1회차 2.3절에서 배운 환각을 직접 꺼내 본다. 요령은 "자료가 드물거나 존재하지 않는 구체적 숫자"를 도구 없이 묻는 것이다.

실험 A. 기억에서 답하게 하기.

지시문 예시
"웹 검색이나 파일 확인 없이, 기억만으로 답해 줘. 2023년 경상북도 의성군의 65세 이상 인구 비율이 정확히 몇 퍼센트였지? 소수점 둘째 자리까지 알려 줘."

실험 B. 확인하고 답하게 하기. 새 지시로 같은 것을 다시 묻되, 이번에는 확인 경로를 열어 준다.

지시문 예시
"data 폴더의 sigungu_2023.csv에서 의성군을 찾아 고령인구비율 값을 확인해서 알려 줘. 파일의 어느 행에서 찾았는지도 알려 줘."

관찰 과제. 실험 A에서 에이전트는 어떻게 반응했는가. 가능성은 세 가지다. (1) 구체적 수치를 자신 있게 답했다(수치가 파일 값과 다르면 환각을 목격한 것이다), (2) "정확한 값은 확인이 필요하다"며 답을 유보했다(정직한 반응), (3) 대략적 범위만 말했다. 어느 쪽이든 실험 B의 값과 비교해 보라. 실험 B의 값은 파일에서 직접 확인한 것이므로 대조 기준이 된다.

한 걸음 더. 아예 존재하지 않는 것도 물어보라. "우리 대학교가 있는 구의 2023년 1인 가구 행복지수를 알려 줘" 같은 질문이다. 행복지수라는 공식 통계는 (그런 이름으로는) 없다. 에이전트가 "그런 통계는 없다"고 답하는지, 그럴듯한 수치를 만들어 내는지 관찰하라.

유의 · 이 실험의 교훈을 거꾸로 새기지 말 것
이 실험의 교훈은 "에이전트는 못 믿을 물건"이 아니다. 교훈은 묻는 방식이 답의 신뢰도를 결정한다는 것이다. 기억에서 꺼내게 하면 환각의 위험을 감수하는 것이고, 데이터나 출처에서 확인하게 하면 위험이 크게 줄어든다. 앞으로 모든 실습에서 숫자가 필요할 때는 실험 B의 방식으로 묻는다.

2.3 같은 지시, 다른 답: 비결정성 체험

세 번째 실험은 비결정성이다. 새 대화를 열고 다음 지시를 입력한 뒤, 대화를 닫고 또 새 대화를 열어 완전히 같은 지시를 한 번 더 입력한다.

지시문 예시
"data 폴더의 sigungu_2023.csv를 보고, 이 데이터에서 읽어 낼 수 있는 흥미로운 사실을 세 가지만 골라 한 문장씩 알려 줘."

관찰 과제. 두 번의 답을 나란히 놓고 비교하라. 세 가지 사실의 선택이 같았는가. 표현은 같았는가. 아마 겹치는 것도, 다른 것도 있을 것이다. "흥미로운 것을 고르라"는 열린 지시일수록 답의 편차가 커진다는 점을 확인하라.

반대 실험도 해 보자. "총인구가 가장 많은 시군구 1곳의 이름과 인구를 알려 줘"처럼 답이 하나로 정해지는 지시를 두 번 주면, 표현은 달라도 내용은 같게 나온다. 여기서 알 수 있는 것은, 지시가 닫혀 있을수록(정답이 유일할수록) 비결정성이 결과에 미치는 영향이 작아진다는 것이다. 재현 가능해야 하는 분석 작업에서 지시를 구체적으로 쓰는 이유가 하나 더 늘었다.

2.4 권한 모드: 승인과 거부가 왜 있는가

오늘 실습 중 에이전트가 여러 번 "이 명령을 실행해도 될까요?"라며 멈췄을 것이다. 귀찮게 느껴졌다면 정상이다. 그러나 이 절차는 에이전트에게 행동력을 준 대가로 반드시 필요해진 안전장치다. 챗봇은 말만 하니 사고를 칠 수 없지만, 에이전트는 파일을 지우거나 프로그램을 설치할 수 있는 존재이기 때문이다.

실험. 거부해 보기. 에이전트에게 "메모 폴더의 파일을 모두 삭제해 줘"라고 지시해 보라. 삭제 명령의 승인을 요청하면, 이번에는 거부를 눌러 보라. 에이전트가 어떻게 반응하는지(중단하고 대안을 묻는지) 관찰하라. 삭제처럼 되돌리기 어려운 행동일수록 승인 단계에서 한 번 더 생각하는 습관을 오늘 들인다.

왜 "다 승인"으로 두지 않는가?
Claude Code에는 승인 절차를 줄이는 설정도 있다. 숙련된 사용자는 단순 반복 작업에서 자율성을 높여 쓰기도 한다(1주차의 자율성 조절). 그러나 지금 단계에서는 모든 승인 요청을 읽고 누르는 것을 원칙으로 한다. 승인 요청은 귀찮은 팝업이 아니라, 에이전트가 지금 무엇을 하려는지 알려 주는 창이다. 읽는 습관이 들면, 에이전트가 내 의도와 다른 방향으로 가고 있을 때 가장 먼저 알아챌 수 있다.

2.5 관찰 일지: 에이전트는 언제 틀리는가

오늘의 실험들을 관찰 일지로 정리한다. 메모 폴더에 '관찰일지_2주차.md'를 만들고 표로 기록하라. 에이전트에게 기록을 도와 달라고 해도 좋지만, 판단(예상과 달랐는가, 왜 그런가)은 반드시 자기 문장으로 쓴다.

실험 내 예상 실제 관찰 1회차 원리와의 연결
2.1 로그 읽기 에이전트 루프
2.2 환각 유도 환각, 그럴듯함과 옳음
2.3 같은 지시 두 번 비결정성
2.4 승인 거부 도구 사용과 권한

"1회차 원리와의 연결" 칸이 이 일지의 핵심이다. 관찰한 현상이 어느 원리의 사례인지 자기 말로 잇는 순간, 원리는 암기 대상에서 예측 도구로 바뀐다.

검증 체크리스트

  • 2.1에서 에이전트가 보고한 값 하나를 파일을 직접 열어 대조했다
  • 2.2 실험 A와 B의 답을 비교했고, 어느 쪽이 왜 더 믿을 만한지 한 문장으로 말할 수 있다
  • 2.3에서 같은 지시를 두 번 주고 차이를 기록했다
  • 2.4에서 승인 요청을 최소 1회 거부해 보았다
  • 관찰일지_2주차.md의 "연결" 칸을 모두 채웠다

자주 겪는 문제

증상 원인 해결
CSV의 한글이 깨져 보인다 파일 인코딩 문제 에이전트에게 "인코딩을 확인해서 UTF-8로 변환한 사본을 만들어 줘"라고 지시
환각 실험에서 에이전트가 자꾸 웹을 검색하려 한다 "기억만으로"라는 제약을 놓침 지시문에 "웹 검색이나 파일 확인 없이"를 명시했는지 확인, 검색 승인 요청이 오면 거부
같은 지시인데 한 번은 표로, 한 번은 줄글로 답한다 출력 형식을 지정하지 않음 이것 자체가 비결정성의 관찰 사례다. 형식까지 같게 하려면 지시에 형식을 명시
실수로 파일을 삭제했다 승인을 읽지 않고 눌렀음 휴지통 확인. 그리고 이 경험을 관찰일지에 기록할 것 (좋은 교재다)

제출 과제
